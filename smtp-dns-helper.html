2. DKIM

את מפתח ה-DKIM חייבים לייצר בקונסולה של Google — הוא ייחודי לכל דומיין.

איך מייצרים:
Admin Console (admin.google.com) ⇐ Apps ⇐ Google Workspace ⇐ Gmail ⇐ Authenticate email ⇐ בחר דומיין ⇐ Generate new record ⇐ 2048-bit ⇐ העתק את ה-TXT.
Type: TXT  |  Host: google._domainkey  |  TTL: 3600
v=DKIM1; k=rsa; p=<public key מה-Console של Google>
לאחר הוספת הרשומה, יש לחזור ל-Admin Console וללחוץ Start authentication.

3. DMARC (מומלץ)

Type: TXT  |  Host: _dmarc  |  TTL: 3600
v=DMARC1; p=quarantine; rua=mailto:dmarc@DOMAIN; adkim=s; aspf=s

יוזר בודד
דומיין מאושר
במצב "יוזר בודד" מאמתים אצל Amazon SES כתובת מייל בודדת. אין צורך בגישה ל-DNS — המשתמש עצמו יאשר את הכתובת דרך לינק שיישלח אליו.

תהליך האימות והמסירה ללקוח

1. רישום הכתובת ב-SES
אנחנו רושמים את כתובת המייל ב-SES Console (Verified identities → Create identity → Email address).

2. הלקוח מקבל מייל אימות
Amazon Web Services שולח אוטומטית הודעה לכתובת שרשמנו, בכותרת:
Amazon Web Services – Email Address Verification Request
הלקוח צריך ללחוץ על הלינק "Verify this email address" בתוך 24 שעות. לאחר הלחיצה מופיע דף אישור באנגלית — זה מסיים את התהליך מבחינתו.

3. יצירת SMTP Credentials
אנחנו יוצרים SMTP Username + Password ב-SES Console (SMTP settings → Create SMTP credentials). AWS מחזיר קובץ CSV עם שני הערכים — הקובץ מוצג פעם אחת בלבד ואי אפשר לשחזר אותו אחר כך.

4. מסירה ללקוח
שולחים ללקוח את שם המשתמש והסיסמה מהקובץ, יחד עם הגדרות ה-SMTP למטה.

עצות מהירות

  • לכל דומיין מותרת רק רשומת SPF אחת. אם קיימת רשומה, יש לשלב את ה-include בתוכה במקום להוסיף חדשה.
  • SPF מוגבל ל-10 lookups. כל include:, a, mx נחשבים כ-lookup.
  • שינויים ב-DNS נכנסים לתוקף לפי ה-TTL של הרשומה הקודמת (עד 24-48 שעות, בפועל דקות בודדות).
  • לבדיקה מהירה: mxtoolbox.com או mail-tester.com.
